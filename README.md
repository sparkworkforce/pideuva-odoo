# Odoo Uva Connector

Odoo module that integrates with [Uva](https://pideuva.com), Puerto Rico's local delivery platform (active since 2017), providing two core integration flows for Odoo POS and delivery operations.

## Flows

### Flow A — Incoming Orders (POS Connector)

Uva customer orders appear automatically in the Odoo POS interface via webhook/polling.
Staff can accept, reject, or mark items as unavailable directly from POS.
Real-time notifications delivered to POS sessions via `bus.bus`.

### Flow B — Outbound Delivery (Uva Fleet)

Dispatch deliveries via Uva Fleet drivers from sale orders or stock pickings.
Extends `delivery.carrier` for native Odoo shipping integration.
Includes cost estimation, order creation, status tracking, and cancellation.

## Supported Versions

| Branch | Odoo Version | Python |
|--------|-------------|--------|
| `17.0` | Odoo 17 Enterprise | 3.10+ |
| `18.0` | Odoo 18 Enterprise | 3.11+ |
| `19.0` | Odoo 19 Enterprise | 3.11+ |

Each version has its own manifest (`__manifest_17__.py`, `__manifest_18__.py`, `__manifest_19__.py`).
Run `./build.sh <17|18|19>` to generate the correct `__manifest__.py` before packaging.

## Requirements

- Odoo Enterprise on [Odoo.sh](https://www.odoo.sh)
- Active Uva / Uva Fleet API credentials
- Odoo modules: `point_of_sale`, `delivery`, `sale_stock`, `mail`, `base_setup`

## Installation

1. Clone the repository into your Odoo addons path:
   ```bash
   git clone https://github.com/sparkworkforce/odoo-uva-connector.git
   ```
2. Build for your Odoo version:
   ```bash
   cd odoo-uva-connector
   ./build.sh 18  # or 17, 19
   ```
3. Update the Odoo module list and install **Uva PR Connector**.

## Configuration

1. Go to **Settings → Uva Connector**
2. Enter your Uva API credentials (stored securely with `groups='base.group_system'`)
3. Configure store mappings in **Uva → Store Configuration**
4. Map Uva products to Odoo products in **Uva → Product Mapping**
5. For Flow B, add **Uva Fleet** as a delivery carrier in **Inventory → Configuration → Shipping Methods**

> **Never hardcode API keys.** All credentials are stored as system-restricted fields.

## Key Features

- Webhook + polling receivers for incoming Uva orders
- Real-time POS notifications via `bus.bus`
- Uva Fleet as a native `delivery.carrier`
- Delivery cost estimation wizard
- Automatic retry queue for failed API calls
- Webhook signature validation
- Chatter integration for order/delivery tracking
- Scheduled cron jobs for polling, retry processing, and payload purge

## Development

odoo-bin -d test_db --test-enable --stop-after-init -i odoo_uva_connector

## Branching Strategy

Development follows the Odoo community convention:
- `17.0` — Odoo 17 stable
- `18.0` — Odoo 18 stable (primary development branch)
- `19.0` — Odoo 19 stable

Features are developed on `18.0` first, then backported/forward-ported as needed.
