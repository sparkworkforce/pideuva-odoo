# Odoo Uva Connector

Odoo module that integrates with [Uva](https://pideuva.com), Puerto Rico's local delivery platform (active since 2017), providing two core integration flows for Odoo POS and delivery operations.

**94 source files · 10,322 lines · 20 models · 9 OWL components · 8 cron jobs**

## Flows

### Flow A — Incoming Orders (POS Connector)

Uva customer orders appear automatically in the Odoo POS interface via webhook/polling.
Staff can accept, reject, modify orders, or mark items as unavailable directly from POS.
Real-time notifications delivered to POS sessions via `bus.bus`.

### Flow B — Outbound Delivery (Uva Fleet)

Dispatch deliveries via Uva Fleet drivers from sale orders or stock pickings.
Extends `delivery.carrier` for native Odoo shipping integration.
Includes cost estimation, order creation, status tracking, delivery proof, and cancellation.

## Features

### Flow A — Incoming Orders

- Webhook + polling receivers for incoming Uva orders
- Order routing rules — auto-assign orders to stores by criteria
- Auto-accept orders via configurable cron
- Order modification support (item changes after acceptance)
- Product auto-mapping with alias support
- Bulk product mapping wizard
- Prep time tracking per order
- Revenue attribution for Uva-sourced orders
- Store hours enforcement

### Flow B — Outbound Delivery

- Uva Fleet as a native `delivery.carrier`
- Delivery cost estimation wizard
- Fleet status polling and webhook updates
- Delivery proof capture
- Delivery zone map with zone widget
- Multi-language customer tracking page
- Customer notifications (email, browser push)

### POS Integration

- OWL-based order popup with accept/reject/modify actions
- Dedicated Uva orders screen in POS
- Health indicator showing API connectivity status
- Error orders panel for failed/problematic orders
- Offline resilience — POS continues working when API is down
- Bus compatibility layer for real-time notifications

### Dashboard

- Per-store dashboard with key metrics
- Performance alerts via cron
- Revenue reporting views
- Analytics views

### Security

- HMAC webhook signature validation
- Rate limiting on webhook endpoints
- XSS prevention in tracking pages
- SSRF protection in API client
- Multi-company isolation (8 `ir.rule` records)
- API credentials restricted to `base.group_system`

## Requirements

- Odoo Enterprise on [Odoo.sh](https://www.odoo.sh)
- Active Uva / Uva Fleet API credentials
- Odoo modules: `point_of_sale`, `delivery`, `sale`, `sale_stock`, `mail`, `base_setup`

## Installation

1. Clone the repository into your Odoo addons path:
   ```bash
   git clone https://github.com/sparkworkforce/odoo-uva-connector.git
   ```
2. Build for your Odoo version:
   ```bash
   cd odoo-uva-connector
   ./build.sh 18  # or 17, 19
   ```
3. Update the Odoo module list and install **Uva PR Connector**.

## Configuration

1. Run the **Setup Wizard** on first install (or go to **Settings → Uva Connector**)
2. Enter your Uva API credentials (stored securely with `groups='base.group_system'`)
3. Configure store mappings in **Uva → Store Configuration** (including store hours and delivery zones)
4. Map Uva products to Odoo products in **Uva → Product Mapping** (or use the Bulk Mapping Wizard)
5. Configure order routing rules in **Uva → Order Rules**
6. For Flow B, add **Uva Fleet** as a delivery carrier in **Inventory → Configuration → Shipping Methods**
7. Review the per-store dashboard in **Uva → Dashboard**

> **Never hardcode API keys.** All credentials are stored as system-restricted fields.

## Branching Strategy

- `main` — primary development branch (targets Odoo 18)
- `17.0` — Odoo 17 stable
- `18.0` — Odoo 18 stable
- `19.0` — Odoo 19 stable

Features are developed on `main` first, then ported to version branches as needed.
